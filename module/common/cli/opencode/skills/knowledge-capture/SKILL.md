# Knowledge Capture

## Overview

Every bug fix, human correction, and project-specific constraint is a learning opportunity. Capture it so future sessions never rediscover the same fix. Periodically review accumulated knowledge to extract general patterns that deserve their own reusable skill.

## When to Capture

### Bug Fixes

Before moving on, capture the root cause, what changed, and how to prevent recurrence.

### Human Corrections

When corrected, capture what you assumed wrong, the correct approach, and what should alert you next time.

### Project Constraints

When discovering an undocumented constraint, capture the limitation, why it exists, and where it applies.

## Behavior

Treat capture as a mandatory step before considering a task done:

1. Complete the task
2. Ask: "Did I learn something this session that future sessions need?"
3. If yes, write one or more memory entries
4. Mark the task done

Do not skip because "it's obvious." The point is persistence, not novelty.

## Periodic Promotion

Periodically review accumulated memories and look for patterns that repeat across projects. When a pattern appears in multiple projects and is general enough to be useful elsewhere, promote it into a dedicated skill or skill update.

This bridges two feedback loops:
- **Short loop**: Capture project-specific knowledge into memory immediately
- **Long loop**: Periodically distill cross-project patterns from memory into reusable skills
