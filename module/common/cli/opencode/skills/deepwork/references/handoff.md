# Phase 5: Handoff

Close the session and ensure continuity.

## Progress File

Update `.slim/deepwork/<slug>.md` with:
- Completed phases and their results
- Unresolved decisions and open questions
- Validation results
- Follow-ups and blockers

## Handoff Document

When work must cross session boundaries, save `.slim/deepwork/handoff-<slug>.md` with:
- Current goal and completed phases
- Unresolved decisions and open questions
- Key research findings with file paths
- Active status and next steps

## Before Finishing

1. Original requirements verified
2. All tests pass
3. Build/typecheck/lint clean
4. Evaluate whether project knowledge (structure, constraints, architecture) was discovered and should be persisted
5. Progress file updated
6. Handoff document written (if crossing sessions)
7. Final summary stated

## When Resuming

Read the handoff document and progress file to restore context before proceeding.
