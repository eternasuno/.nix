# Micro Capture

Capture learnings from bugs, corrections, and constraints after every significant task.

## When to Capture

### Bug Fixes
Before moving on, capture: root cause, what changed, how to prevent recurrence.

### Human Corrections
When corrected, capture: what you assumed wrong, the correct approach, what should alert you next time.

### Project Constraints
When discovering an undocumented constraint: the limitation, why it exists, where it applies.

## Behavior

1. Complete the task
2. Ask: "Did I learn something this session that future sessions need?"
3. If yes, write one or more memory entries
4. Mark the task done

Do not skip because "it's obvious." The point is persistence, not novelty.

## Storage

Use `ctx_memory` to write durable project memories. Categorize as:
- PROJECT_RULES — workflows and conventions
- ARCHITECTURE — structural facts
- CONSTRAINTS — limitations and restrictions
- CONFIG_VALUES — settings and paths
- NAMING — naming conventions
