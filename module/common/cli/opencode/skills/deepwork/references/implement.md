# Phase 2: Implement

Execute implementation phases with discipline.

## TDD Loop

For each phase:
1. Write the failing test first
2. Write only enough code to pass it
3. One seam, one test, one minimal implementation per cycle

## Simplicity Constraint

- No features beyond what was asked
- No abstractions for single-use code
- Match existing style

## Surgical Discipline

- Touch only what the phase requires
- Don't refactor adjacent code
- Clean up only your own orphans (imports, variables made unused by your changes)

## Designer Handoff Guardrail

When a phase includes designer work:
- Preserve layout, rhythm, hierarchy, motion, spacing, color, affordances
- Route visual/polish follow-ups back to designer
- Use fixer only for mechanical follow-up that preserves design exactly

## Output

- Working code with passing tests
- No scope creep
- Update progress file with phase result
